# AI Context: Dictionary Backend

## Что это за проект

Это backend для **offline-first словаря с нейросетями**.

Основной клиент — **мобильное приложение на Expo**.  
В будущем приложение также должно работать **в вебе**, используя тот же backend.

---

# Цель проекта

Основная задача — помогать людям изучать слова иностранного языка, **генерируя контекст для них**, чтобы:

- было интереснее их учить
- было легче запоминать
- пользователь сразу понимал, **где и как использовать слова**

Пользователь добавляет слова в виде **карточек**, после чего приложение может генерировать обучающий контент на основе этих карточек.

---

# Архитектура

Приложение использует **offline-first подход**.

### Клиент

Основные данные хранятся **локально на устройстве пользователя** при помощи **SQLite**.

Локально хранятся:

- карточки
- тренировочные пресеты
- результаты тренировок
- истории
- другие пользовательские данные

Клиент должен **работать без сети**, кроме AI функционала.

Основная платформа — **мобильные устройства**, но приложение также должно работать **в вебе**.

---

### Backend

Backend выполняет следующие задачи:

- синхронизация данных между клиентом и сервером
- хранение пользовательских данных
- работа с нейросетями
- авторизация и регистрация пользователей

Backend **не является единственным источником истины**, потому что клиент должен продолжать работать **в offline режиме**.

---

# Работа с нейросетями

Нейросети используются для генерации обучающего контента на основе карточек пользователя.

Основные функции:

### 1. Генерация тренировочных упражнений

Создание упражнений на основе карточек пользователя.

### 2. Генерация интерактивных историй

Создание историй, в которых используются слова пользователя.

Истории могут содержать:

- контекст использования слов
- несколько вариантов развития событий
- выбор действий пользователем

---

# Безопасность AI

API ключи нейросетей **никогда не должны находиться на клиенте**.

Все запросы к нейросетям выполняются **через backend**.

Архитектура запросов:
client → backend → AI provider

Используемые AI провайдеры:

- Gemini
- OpenRouter

Пользователи **не используют собственные AI ключи**.

Backend использует **серверные ключи** и может управлять:

- лимитами
- usage
- кэшированием ответов

3. Хранение карточек пользователей

# Принципы разработки

Backend должен:

- поддерживать offline-first архитектуру
- не требовать постоянного соединения с сервером
- не раскрывать API ключи
- быть модульным
- быть простым для расширения
- избегать излишней сложности на ранних этапах

# What AI agents should keep in mind

When modifying this project:

- preserve offline-first assumptions
- never expose AI API keys to clients
- keep architecture simple
- prefer modular NestJS structure
- avoid unnecessary abstractions
